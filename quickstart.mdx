## Prerequisites

Before you begin, ensure you have the following installed and configured:

- **Node.js** (v16 or higher recommended)
- **MongoDB** (local installation or MongoDB Atlas account)
- **Cloudinary Account** (free tier available at [cloudinary.com](https://cloudinary.com))
- **Package Manager** (npm, yarn, or pnpm)

<Note>
  The API uses TypeScript and requires Node.js to compile and run. MongoDB can be running locally or you can use a cloud database like MongoDB Atlas.
</Note>

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the Dev-Tree backend repository to your local machine:
    
    ```bash
    git clone https://github.com/HeymerDev/dev-tree-backend.git
    cd dev-tree-backend
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages using your preferred package manager:
    
    <CodeGroup>
    ```bash npm
    npm install
    ```
    
    ```bash yarn
    yarn install
    ```
    
    ```bash pnpm
    pnpm install
    ```
    </CodeGroup>
    
    This will install all dependencies including:
    - Express, TypeScript, and type definitions
    - MongoDB driver (Mongoose)
    - Authentication libraries (bcrypt, jsonwebtoken)
    - Cloudinary SDK for image uploads
    - Validation and parsing utilities
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the root directory with the following variables:
    
    ```bash
    # Server Configuration
    PORT=3000
    
    # MongoDB Connection
    MONGO_URI=mongodb://localhost:27017/dev-tree
    
    # Cloudinary Configuration
    CLOUD_NAME=your_cloudinary_cloud_name
    CLOUDINARY_API_KEY=your_cloudinary_api_key
    CLOUDINARY_API_SECRET=your_cloudinary_api_secret
    
    # JWT Secret
    JWT_SECRET=your_super_secret_jwt_key_change_this
    
    # CORS Configuration
    FRONTEND_URL=http://localhost:5173
    ```
    
    <Warning>
      Make sure to replace the placeholder values with your actual credentials. Never commit the `.env` file to version control.
    </Warning>
    
    **Getting your credentials:**
    
    - **MONGO_URI**: For local MongoDB, use `mongodb://localhost:27017/dev-tree`. For MongoDB Atlas, get your connection string from the Atlas dashboard.
    - **Cloudinary credentials**: Log into your Cloudinary account and find these in your dashboard under "Account Details".
    - **JWT_SECRET**: Generate a secure random string. This is used to sign authentication tokens.
    - **FRONTEND_URL**: The URL of your frontend application for CORS configuration.
  </Step>

  <Step title="Start the development server">
    Run the development server with hot-reload enabled:
    
    <CodeGroup>
    ```bash npm
    npm run dev
    ```
    
    ```bash yarn
    yarn dev
    ```
    
    ```bash pnpm
    pnpm dev
    ```
    </CodeGroup>
    
    The server will start on `http://localhost:3000` (or your configured PORT).
    
    <Note>
      Use `npm run dev:api` to enable API testing tools (allows requests from Postman, Insomnia, etc. without CORS restrictions).
    </Note>
  </Step>
</Steps>

## Make Your First API Request

Let's register a new user to verify the API is working correctly.

### Register a New User

Send a POST request to create your first user account:

```bash cURL
curl -X POST http://localhost:3000/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "handle": "johndoe",
    "username": "John Doe",
    "email": "john@example.com",
    "password": "securepass123"
  }'
```

**Expected Response:**

```json
{
  "message": "User created successfully"
}
```

<Note>
  The `handle` will be automatically converted to a URL-friendly slug (lowercase, hyphenated). For example, "John Doe" becomes "john-doe".
</Note>

### Login and Get JWT Token

Authenticate with your new account to receive a JWT token:

```bash cURL
curl -X POST http://localhost:3000/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "john@example.com",
    "password": "securepass123"
  }'
```

**Expected Response:**

```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

### Get User Profile by Handle

Retrieve any user's public profile using their handle:

```bash cURL
curl http://localhost:3000/johndoe
```

**Expected Response:**

```json
{
  "handle": "johndoe",
  "username": "John Doe",
  "description": "",
  "imageUrl": "",
  "links": "[]"
}
```

<Note>
  Notice that sensitive fields like `password`, `email`, and `_id` are excluded from public profile responses.
</Note>

## Available NPM Scripts

The following scripts are available in `package.json`:

| Script | Command | Description |
|--------|---------|-------------|
| **Development** | `npm run dev` | Start development server with hot-reload |
| **Development (API mode)** | `npm run dev:api` | Start dev server with relaxed CORS for API testing |
| **Build** | `npm run build` | Compile TypeScript to JavaScript in `dist/` folder |
| **Production** | `npm start` | Run the compiled production build |
| **Test** | `npm test` | Run tests (not configured yet) |

## Project Structure

Here's what you'll find in the source code:

```
src/
├── config/
│   ├── db.ts              # MongoDB connection setup
│   ├── cloudinary.ts      # Cloudinary SDK configuration
│   └── cors.ts            # CORS policy configuration
├── controllers/
│   └── index.ts           # Request handlers (register, login, etc.)
├── middlewares/
│   ├── auth.ts            # JWT authentication middleware
│   └── middleware.ts      # Input validation error handling
├── models/
│   └── User.ts            # User schema and model
├── utils/
│   ├── auth.ts            # Password hashing utilities
│   ├── jwt.ts             # JWT generation utilities
│   └── loadEnv.ts         # Environment variable loader
├── router.ts              # Route definitions and validation
├── server.ts              # Express app configuration
└── index.ts               # Entry point and server startup
```
