<Note>
This endpoint requires JWT authentication. Include the JWT token in the `Authorization` header as a Bearer token.
</Note>

## Endpoint

```
POST /user/image
```

This endpoint allows authenticated users to upload a profile image. The image is stored in Cloudinary under the "dev-tree" folder with a unique UUID-based identifier.

## How It Works

The upload process follows these steps:

1. **Authentication**: The request is validated using JWT authentication middleware
2. **Multipart Parsing**: The endpoint uses `formidable` to parse multipart/form-data
3. **Cloudinary Upload**: The file is uploaded to Cloudinary with:
   - Folder: `dev-tree`
   - Public ID: Generated using UUID v4
4. **Database Update**: The user's `imageUrl` field is updated with the Cloudinary secure URL
5. **Response**: Returns the uploaded image URL

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token containing the JWT for the authenticated user.
  
  Format: `Bearer <jwt_token>`
</ParamField>

## Request

### Content Type

```
multipart/form-data
```

### Form Fields

<ParamField body="file" type="file" required>
  The image file to upload. This should be sent as a multipart form field.
</ParamField>

<Warning>
While the API accepts image uploads, ensure you handle file size and type validation on the client side. Consider limiting file sizes to prevent excessive upload times and storage costs.
</Warning>

## Response

### Success Response (200)

<ResponseField name="imageUrl" type="string">
  The secure URL of the uploaded image on Cloudinary.
</ResponseField>

<ResponseField name="message" type="string">
  Success message confirming the upload.
</ResponseField>

```json
{
  "imageUrl": "https://res.cloudinary.com/your-cloud/image/upload/v1234567890/dev-tree/uuid-here.jpg",
  "message": "Image uploaded successfully"
}
```

### Error Response (500)

```json
{
  "message": "Internal server error"
}
```

## Implementation Details

### Upload Controller

The `uploadImage` function from `src/controllers/index.ts` handles the upload:

```typescript
export const uploadImage = async (req: Request, res: Response) => {
  const form = formidable({ multiples: false });

  try {
    form.parse(req, (err, fields, files) => {
      console.log(files.file[0].filepath);

      cloudinary.uploader.upload(
        files.file[0].filepath,
        {
          folder: "dev-tree",
          public_id: uuid(),
        },
        async function (error, result) {
          if (error) {
            const newError = new Error("Subida de imagen fallida");
            res.status(500).json({ message: "Internal server error" });
          }
          if (result) {
            req.user.imageUrl = result.secure_url;
            await req.user.save();
            res.status(200).json({
              imageUrl: result.secure_url,
              message: "Image uploaded successfully",
            });
          }
        }
      );
    });
  } catch (error) {
    new Error(error);
    res.status(500).json({ message: "Internal server error" });
  }
};
```

### Cloudinary Configuration

The Cloudinary service is configured in `src/config/cloudinary.ts`:

```typescript
import { v2 as cloudinary } from "cloudinary";

cloudinary.config({
  cloud_name: process.env.CLOUD_NAME,
  api_key: process.env.CLOUDINARY_API_KEY,
  api_secret: process.env.CLOUDINARY_API_SECRET,
});

export default cloudinary;
```

### Formidable Usage

The endpoint uses `formidable` to parse multipart/form-data:

- **multiples: false** - Only accepts a single file upload
- The file is accessed via `files.file[0].filepath`
- The temporary file path is passed directly to Cloudinary for upload

### Cloudinary Folder Structure

All uploaded images are stored in the `dev-tree` folder on Cloudinary with:
- **Folder**: `dev-tree` - Organizes all Dev-Tree profile images
- **Public ID**: UUID v4 - Ensures unique identifiers for each upload

## Code Examples

### cURL

```bash
curl -X POST https://api.example.com/user/image \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -F "file=@/path/to/image.jpg"
```

### JavaScript (fetch)

```javascript
const uploadProfileImage = async (file, token) => {
  const formData = new FormData();
  formData.append('file', file);

  const response = await fetch('https://api.example.com/user/image', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`
    },
    body: formData
  });

  const data = await response.json();
  
  if (response.ok) {
    console.log('Image uploaded:', data.imageUrl);
    return data;
  } else {
    throw new Error(data.message);
  }
};

// Usage
const fileInput = document.querySelector('input[type="file"]');
const file = fileInput.files[0];
const token = 'your-jwt-token';

uploadProfileImage(file, token)
  .then(data => console.log('Success:', data))
  .catch(error => console.error('Error:', error));
```

### JavaScript (FormData with file input)

```javascript
// HTML: <input type="file" id="imageUpload" accept="image/*">

document.getElementById('imageUpload').addEventListener('change', async (e) => {
  const file = e.target.files[0];
  
  if (!file) return;
  
  const formData = new FormData();
  formData.append('file', file);
  
  try {
    const response = await fetch('/user/image', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${localStorage.getItem('token')}`
      },
      body: formData
    });
    
    const result = await response.json();
    
    if (response.ok) {
      console.log('Image URL:', result.imageUrl);
      // Update UI with new image
      document.getElementById('profileImage').src = result.imageUrl;
    } else {
      console.error('Upload failed:', result.message);
    }
  } catch (error) {
    console.error('Error uploading image:', error);
  }
});
```

## Route Configuration

From `src/router.ts`:

```typescript
router.post("/user/image", authenticate, uploadImage);
```

The route is protected by the `authenticate` middleware, which validates the JWT token and attaches the user object to the request.
