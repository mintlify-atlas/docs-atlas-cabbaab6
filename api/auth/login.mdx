## POST /auth/login

Authenticates a user with their email and password. Upon successful authentication, returns a JWT (JSON Web Token) that can be used to access protected endpoints.

The JWT token should be included in the `Authorization` header as a Bearer token for subsequent authenticated requests.

### Request Body

<ParamField path="email" type="string" required>
  User's registered email address.
  
  **Validation:** Must be a valid email format (e.g., user@example.com)
</ParamField>

<ParamField path="password" type="string" required>
  User's password.
  
  **Validation:** Cannot be empty
</ParamField>

### Response

<ResponseField name="token" type="string">
  JWT authentication token to be used for accessing protected endpoints. Include this token in the Authorization header as `Bearer <token>`.
</ResponseField>

<ResponseField name="message" type="string">
  Error message when authentication fails.
</ResponseField>

### Status Codes

- **200 OK** - Authentication successful, JWT token returned
- **400 Bad Request** - Validation errors (invalid email format or empty password)
- **401 Unauthorized** - Invalid password provided
- **404 Not Found** - No user found with the provided email (returned as "Invalid credentials" for security)

### Examples

<CodeGroup>

```bash curl
curl -X POST https://api.devtree.com/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "john@example.com",
    "password": "securepass123"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.devtree.com/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    email: 'john@example.com',
    password: 'securepass123'
  })
});

const data = await response.json();

// Store the token for future requests
if (response.ok) {
  localStorage.setItem('authToken', data.token);
}

console.log(data);
```

```python Python
import requests

response = requests.post(
    'https://api.devtree.com/auth/login',
    json={
        'email': 'john@example.com',
        'password': 'securepass123'
    }
)

data = response.json()

# Store the token for future requests
if response.status_code == 200:
    token = data['token']
    print(f'Login successful. Token: {token}')
```

</CodeGroup>

### Validation Rules

The following validation rules are enforced (from `router.ts:17-20`):

1. **email**: Must be a valid email format (validated using `isEmail()` from express-validator)
2. **password**: Must not be empty

### Using the JWT Token

Once you receive the JWT token, include it in the `Authorization` header for protected endpoints:

<CodeGroup>

```bash curl
curl -X GET https://api.devtree.com/user \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

```javascript JavaScript
const token = localStorage.getItem('authToken');

const response = await fetch('https://api.devtree.com/user', {
  method: 'GET',
  headers: {
    'Authorization': `Bearer ${token}`
  }
});

const userData = await response.json();
```

```python Python
token = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."

response = requests.get(
    'https://api.devtree.com/user',
    headers={'Authorization': f'Bearer {token}'}
)

user_data = response.json()
```

</CodeGroup>

### Implementation Details

From `controllers/index.ts:34-54`, the login process:

1. Checks if a user exists with the provided email
   - If not found, returns 404 with "Invalid credentials" message
2. Compares the provided password with the stored hashed password using bcrypt
   - If password doesn't match, returns 401 with "Invalid credentials" message
3. Generates a JWT token containing the user's ID
4. Returns the token in the response

<Info>
  Both "user not found" and "invalid password" scenarios return "Invalid credentials" to prevent user enumeration attacks.
</Info>

<Note>
  The JWT token contains the user's ID and can be decoded to retrieve user information in protected routes via the `authenticate` middleware.
</Note>

<Warning>
  Store JWT tokens securely (e.g., httpOnly cookies or secure storage). Never expose tokens in URLs or log them in plain text.
</Warning>
